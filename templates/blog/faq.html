What is this?

While studying SQL for a technical interview, I found myself spending substantially more time creating and populating tables than actually learning how to JOIN and SELECT data (what I should have been learning).
SQL Butler will take any table and generate the appropriate CREATE and INSERT statements to execute in mysql.

How do I use this?

Find tables on SQL interview prep blogs, tables, create your own in Excel, and paste them in to generate sql statements to create your own tables.

Why is everything a varchar(150)?

Edit your own damn data types. Just kidding, I figured if you're using this to practice joining tables, the data types don't really matter that much. Plus, the create statement isn't that bad, it's the insert statements that suck. Plus, I didn't want to bog down the interface with an array of drop downs (pretty good excuse, right?).
