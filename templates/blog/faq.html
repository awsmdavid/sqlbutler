What is this?

SQLButler.com is an online SQL statement assistant that will generate a CREATE and INSERT statement from any table.

Why would anyone use this?

While studying up on SQL for a technical interview, I found myself spending significantly more time creating and populating practice tables than actually learning how to JOIN and SELECT data (what the actual interviewers cared about).

s to generate tables so that you can spend more time practicing and less time doing data entry... and improve your SQL!

How do I use this?

Boot up your favorite instance of SQL (e.g., mysql), create a practice DB, and use these Copy and paste any tables you find on SQL interview prep blogs, forums, or tables you create in Excel, and paste them in the input field to generate SQL statements to generate CREATE and INSERT statements.

Why is everything a varchar(150)?

Edit your own damn data types. Just kidding, I figured if you're using this to practice joining tables, the data types don't really matter. Plus, the create statement isn't the really tedious command, it's the insert statements that suck. And from a UI perspective, I didn't want to bog down the input area with an array of drop downs (pretty good excuses, right?).
